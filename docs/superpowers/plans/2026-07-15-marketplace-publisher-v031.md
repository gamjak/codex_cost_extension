# Marketplace Publisher v0.3.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish a CI-verified v0.3.1 package whose manifest publisher matches the Marketplace publisher account `gamjak`.

**Architecture:** The extension manifest remains the source of truth for Marketplace identity. A raw-manifest regression assertion prevents future publisher drift, while the existing CI package job proves the resulting VSIX can be built and validated before release.

**Tech Stack:** VS Code extension manifest, TypeScript/Vitest, pnpm, GitHub Actions, VSCE.

## Global Constraints

- Set `publisher` exactly to `gamjak`.
- Set package version exactly to `0.3.1`.
- Preserve GitHub release `v0.3.0`; do not alter its assets or tag.
- Do not automatically publish to the VS Code Marketplace.
- Attach only the VSIX generated by a passing CI package job to the `v0.3.1` GitHub release.

### Task 1: Correct the Marketplace identity and release metadata

**Files:**
- Modify: `package.json`
- Modify: `CHANGELOG.md`
- Modify: `test/unit/packageManifest.test.ts`

**Interfaces:**
- Consumes: `package.json` manifest fields `publisher` and `version`.
- Produces: Marketplace extension identifier `gamjak.codex-cost-extension` and release version `0.3.1`.

- [ ] **Step 1: Write the failing manifest assertion**

Add the expected publisher and version to `test/unit/packageManifest.test.ts`:

```ts
expect(manifest.publisher).toBe('gamjak');
expect(manifest.version).toBe('0.3.1');
```

- [ ] **Step 2: Run the focused test to verify it fails**

Run: `pnpm vitest run test/unit/packageManifest.test.ts`

Expected: failure because the current manifest contains `gambjako` and `0.3.0`.

- [ ] **Step 3: Apply the minimal manifest and changelog change**

In `package.json` set:

```json
"version": "0.3.1",
"publisher": "gamjak"
```

Add this top entry to `CHANGELOG.md`:

```markdown
## 0.3.1 - 2026-07-15

- Correct the Visual Studio Marketplace publisher ID to `gamjak`.
```

- [ ] **Step 4: Run the focused test and package contract**

Run:

```bash

pnpm run check

```

Expected: all commands exit 0. If the local environment cannot provision dependencies, record the exact blocker and rely on the unchanged GitHub CI matrix before release.

- [ ] **Step 5: Commit the change**


### Task 2: Validate, merge, and release v0.3.1

**Files:**
- No source-file changes.

**Interfaces:**
- Consumes: the Task 1 manifest and the existing CI package artifact.
- Produces: merged `main`, annotated tag `v0.3.1`, GitHub release asset `codex-cost-extension.vsix`.

- [ ] **Step 1: Push and create a pull request**

```bash
git push --set-upstream origin fix/marketplace-publisher-v031
gh pr create --base main --head fix/marketplace-publisher-v031 --title "Publish v0.3.1 under gamjak" --body "Corrects the Marketplace publisher ID and creates a clean v0.3.1 release."
```

- [ ] **Step 2: Verify required checks**

Set `$PR_NUMBER` to the pull request number returned in Step 1, then run: `gh pr checks $PR_NUMBER --watch`.

Expected: `package`, `validate (ubuntu-latest)`, `validate (macos-latest)`, and `validate (windows-latest)` all pass.

- [ ] **Step 3: Merge only after all checks pass**

```bash
gh pr merge $PR_NUMBER --merge --delete-branch
```

Expected: the merge commit becomes the `main` target for v0.3.1.

- [ ] **Step 4: Download and hash the passing VSIX artifact**

```bash
gh run download $RUN_ID --name codex-cost-extension --dir work/release-assets-v031
Get-FileHash work/release-assets-v031/codex-cost-extension.vsix -Algorithm SHA256
```

Set `$RUN_ID` to the successful CI run ID captured from `gh pr checks`. Expected: one VSIX file and a recorded SHA-256 digest.

- [ ] **Step 5: Create the annotated tag and GitHub release**

```bash
git fetch origin main
git tag -a v0.3.1 origin/main -m "Release v0.3.1"
git push origin v0.3.1
gh release create v0.3.1 work/release-assets-v031/codex-cost-extension.vsix --title "Codex Cost 0.3.1" --notes "Corrects the Marketplace publisher ID to gamjak. Marketplace publication is manual."
```

Expected: a non-draft GitHub release with the CI-built VSIX asset; no Marketplace publication occurs.

- [ ] **Step 6: Verify release identity and asset**

```bash
gh release view v0.3.1 --json tagName,targetCommitish,isDraft,assets
```

Expected: tag `v0.3.1`, target `main`, `isDraft: false`, and an uploaded `codex-cost-extension.vsix` asset.
