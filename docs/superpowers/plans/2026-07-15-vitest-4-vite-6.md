# Vitest 4 and Vite 6 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade the test runner to Vitest 4 while resolving its Vite 6 peer dependency so CI and VSIX packaging pass.

**Architecture:** The package manifest becomes the explicit source of the Vitest–Vite compatibility contract. A focused tooling-configuration unit test asserts that contract before the manifest and pnpm lockfile are updated. No extension runtime code or TypeScript configuration changes.

**Tech Stack:** Node.js 22, pnpm 11.7.0, Vitest 4.1.10, Vite 6.x, TypeScript.

## Global Constraints

- Change only `package.json`, `pnpm-lock.yaml`, and `test/unit/toolingConfiguration.test.ts`.
- Pin `vitest` to `^4.1.10` and add direct dev dependency `vite` with `^6.0.0`.
- Do not change extension source, test behavior, or TypeScript configuration.
- Use `pnpm` from the repository's declared `packageManager` field.
- Validation must include `pnpm run check`, VSIX packaging, and `pnpm run verify-package`.

### Task 1: Encode the required Vitest–Vite compatibility contract

**Files:**
- Modify: `test/unit/toolingConfiguration.test.ts`
- Modify: `package.json`
- Modify: `pnpm-lock.yaml`

**Interfaces:**
- Consumes: `readManifest()` in `test/unit/toolingConfiguration.test.ts`, which parses `package.json`.
- Produces: a manifest test that requires explicit compatible dev dependencies.

- [ ] **Step 1: Write the failing manifest test**

Extend `ToolingManifest` with an optional `devDependencies` property and add this test after `separates runtime compilation from type checking and tests`:

```ts
  it('declares a Vite version compatible with the Vitest runner', () => {
    const manifest = readManifest();

    expect(manifest.devDependencies).toMatchObject({
      vitest: '^4.1.10',
      vite: '^6.0.0'
    });
  });
```

Use this interface shape:

```ts
interface ToolingManifest {
  scripts: {
    compile: string;
    check: string;
    test: string;
    watch: string;
    'vscode:prepublish': string;
  };
  devDependencies?: Record<string, string>;
}
```

- [ ] **Step 2: Run the focused test to verify it fails**

Run: `pnpm exec vitest run test/unit/toolingConfiguration.test.ts`

Expected: the new test fails because `vitest` is `^2.1.8` and no direct `vite` dependency exists.

- [ ] **Step 3: Update the manifest and regenerate the lockfile**

In `package.json`, replace the existing Vitest development dependency and add Vite next to it:

```json
"vite": "^6.0.0",
"vitest": "^4.1.10"
```

Run: `pnpm install`

Expected: `pnpm-lock.yaml` records Vitest 4.1.10 and a Vite 6.x resolution; it must no longer resolve Vitest 4 against Vite 5.

- [ ] **Step 4: Run the focused test to verify it passes**

Run: `pnpm exec vitest run test/unit/toolingConfiguration.test.ts`

Expected: PASS, including `declares a Vite version compatible with the Vitest runner`.

- [ ] **Step 5: Commit the compatibility migration**

```powershell
git add package.json pnpm-lock.yaml test/unit/toolingConfiguration.test.ts
git commit -m "build: align Vite with Vitest 4"
```
